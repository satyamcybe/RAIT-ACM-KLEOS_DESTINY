# PRAMAAN — LAYER 1 COMPLETE FRONTEND FLOW

## Context

You are building the complete Layer 1 identity verification user journey for Pramaan.

Pramaan is an AI-powered trust and reputation infrastructure for India's gig workers.
Layer 1 is the identity verification layer — it connects the user's DigiLocker account,
retrieves their eShram card, parses their UAN, cross-validates identity data, and
generates a verified worker profile.

This is a FRONTEND-ONLY build. All API calls are to placeholder route handlers.
Do NOT implement backend logic. Do NOT call real DigiLocker or eShram APIs.
Use mock/sandbox responses where needed.

---

## Architecture Flow (implement exactly in this order)

Landing Page
  → Generate Passport CTA
  → /onboarding (Layer 1 entry)
  → DigiLocker Login screen
  → OAuth redirect (sandbox)
  → /api/digilocker/callback (receives token)
  → /identity/processing (live progress screen)
      Step 1: Fetch user details
      Step 2: Fetch document listing
      Step 3: Find eShram card
      Step 4: Download eShram PDF
      Step 5: Parse UAN
      Step 6: eShram sandbox verify
      Step 7: Cross validation
      Step 8: Generate worker profile
  → /identity/complete (Layer 1 success screen)
  → Proceed to Layer 2 CTA

## State Management

Use React useState and useEffect only. No external state library.

The processing page manages an array of ProcessingStep objects.
Each step's status updates via a sequential async chain:

  const runPipeline = async () => {
    for (let i = 0; i < steps.length; i++) {
      setSteps(prev => prev.map((s, idx) =>
        idx === i ? { ...s, status: 'active' } : s
      ))
      await mockStep(i)   // calls the corresponding mock function
      setSteps(prev => prev.map((s, idx) =>
        idx === i ? { ...s, status: 'complete', detail: getDetail(i) } : s
      ))
    }
    await delay(800)
    router.push('/identity/complete')
  }

Run runPipeline() in useEffect on mount.
